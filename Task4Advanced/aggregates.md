# Агрегаты bounded contexts

## Booking — «Запись»

| Поле | Значение |
|------|----------|
| **Корень** | `Запись` (Booking) |
| **Ключ** | `bookingId` |
| **Граница** | Жизненный цикл записи на визит: создание → ожидание оплаты → подтверждение |

**Инварианты**

- Нельзя подтвердить запись, пока не получено интеграционное событие `Оплата` от контекста Payment.
- Одна активная запись на слот/врача/время (без двойного бронирования).
- Счёт на визит выставляется не более одного раза на запись.

**Связанные сущности (внутри BC)** | Счёт на оплату визита — ссылка `invoiceId`, без владения финансовой логикой.

---

## Payment — «Счёт»

| Поле | Значение |
|------|----------|
| **Корень** | `Счёт` (Invoice) |
| **Ключ** | `invoiceId` |
| **Граница** | Выставление и погашение счёта за медицинскую услугу |

**Инварианты**

- Счёт всегда привязан к `bookingId` (или эквивалентному бизнес-ключу визита).
- Повторная оплата уже оплаченного счёта запрещена (идемпотентность по `paymentId`).
- Сумма и валюта неизменяемы после выставления (корректировка — новый счёт/сторно).

---

## Patient Care — «Визит», «История болезни»

### Агрегат «Визит»

| Поле | Значение |
|------|----------|
| **Корень** | `Визит` (Visit) |
| **Ключ** | `visitId` |
| **Граница** | Один приём: от старта до завершения |

**Инварианты**

- Визит можно начать только после `Подтверждение записи` из Booking.
- Нельзя завершить визит без фиксации итога (диагноз / план / отказ).
- Назначение исследования допустимо только в статусе «визит активен».

### Агрегат «История болезни»

| Поле | Значение |
|------|----------|
| **Корень** | `История болезни` (MedicalRecord) |
| **Ключ** | `patientId` |
| **Граница** | Долгоживущая клиническая карта пациента |

**Инварианты**

- Диагнозы и назначения лечения версионируются; удаление — только пометка, не физическое стирание (мед. и юр. требования).
- Результаты исследований прикрепляются к визиту и дублируются ссылкой в истории.
- Изменения после лечения фиксируются как новая версия плана, не перезапись без аудита.

## Research — «Материал»

| Поле | Значение |
|------|----------|
| **Корень** | `Материал` (Specimen / StudyMaterial) |
| **Ключ** | `researchOrderId` / `materialId` |
| **Граница** | Заказ исследования: образец/снимок → лаборатория → результат |

**Инварианты**

- Исследование материала нельзя начать, пока материал не в статусе «получен лабораторией».
- Один заказ исследования связан с `visitId` и `patientId` из Patient Care.
- Результат публикуется один раз; повторная передача — с версией/коррекцией.

---

## Связи агрегатов между BC (только по ID, без shared DB)

```
Booking.bookingId  ──►  Payment (invoice)
Booking.bookingId  ──►  Patient Care (visit)
Patient Care.visitId  ──►  Research (order)
```
